# ハッカソン版の実装制約

- 開発期間: 2週間
- 開発者: 1人
- 開発支援: AIを活用
- 目的: ハッカソンで動作するMVPを完成・公開する
- 技術基盤: 既存のTanStack Start、React、TypeScript、MantineおよびVite+を維持する
- 配置方針: 可能な限りCloudflareを利用し、低コストに抑える
- データ方針: ハッカソン期間では固定データスナップショットを使用する

## 計画上の原則

- 公開データに基づく地点調査と、1〜3地点の比較を最優先する。
- データ状態と根拠情報を正しく区別できない場合、見た目の完成を優先して推測値を表示しない。
- 将来機能のための空画面、認証、永続データベースおよび定期更新基盤を作らない。
- 外部サービスや既存の公式配信を活用し、ハッカソン内で全国GIS基盤を一から構築しない。
- 必須機能が完成する前に、拡張機能の実装へ進まない。
- 外部APIのレート制限は必要だが、主要導線が動作した後の終盤タスクとし、初期開発を妨げない。
