# ADR-0017: 公開GISデータはR2へ保存する

- 状態: 採用
- 決定日: 2026-07-17

## コンテキスト

ハッカソン版では、関東のA31a、A53および東京都地域危険度という、読み取り中心の固定GISデータを扱う。大容量ポリゴンの地図配信と地点検索が必要だが、ユーザーアカウントやサーバー上の履歴は扱わない。

D1はSQLite互換のSQLデータベースだが、ハッカソン版で使用できることを前提にできる空間拡張はなく、GISポリゴンを格納する場合は独自の空間キー、候補検索、データ投入および版移行が必要になる。

## 決定

- 公開GISデータ本体はCloudflare R2へ保存する。
- 地点検索用データは、空間インデックスとRange読み取りに対応するFlatGeobufへ変換する。
- 地図表示用データはPMTilesへ変換する。
- R2には、データマニフェスト、カバレッジ情報、チェックサムおよび版付きの変換済み成果物を配置する。
- 元のZIPおよび変換途中のデータは公開せず、ブラウザには必要な加工済み成果物だけを配信する。
- 版付きURLと長期間のHTTPキャッシュを使用し、データ版を変更するときは新しいURLを発行する。
- ハッカソン版ではD1を導入しない。

## R2内の役割

- FlatGeobuf: ピン地点判定、重複ポリゴン、周辺境界および根拠情報の取得
- PMTiles: MapLibreによる災害レイヤーの表示
- マニフェスト: データセット名、提供者、基準年度、取得日、利用条件、アプリ内データ版、変換処理版および成果物URL
- カバレッジ情報: 関東の対応範囲、A53の収録水系と降雨規模および東京都地域危険度の対象範囲
- チェックサム: アップロードまたはキャッシュ破損の検出

## 将来のD1利用

D1は、次の動的なユーザーデータが必要になった時点で導入を検討する。

- ユーザーアカウント
- 重視条件
- 保存した調査・比較履歴
- 期限付きまたは取り消し可能な共有リンク
- 詳細なユーザーフィードバック
- データ更新ジョブの実行状態

## 結果

- ハッカソン版の実行時には、公式GIS配信元へ地点ごとのリクエストを行わない。
- ブラウザまたはWorkerはR2から必要なバイト範囲だけを取得できる。
- データ加工スクリプトは、FlatGeobuf、PMTiles、マニフェスト、カバレッジ情報およびチェックサムを同じ版として生成する必要がある。
- 地点判定用と地図用で成果物は分かれるが、同じ正規化済み入力と属性定義から生成して不整合を防ぐ。
- FlatGeobufまたはPMTilesの技術検証が成立しない場合でもD1へ切り替えず、まず成果物の分割単位または地図表示範囲を縮小する。
