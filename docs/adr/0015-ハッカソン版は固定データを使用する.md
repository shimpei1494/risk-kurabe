# ADR-0015: ハッカソン版は固定データを使用する

- 状態: 採用
- 決定日: 2026-07-17

## コンテキスト

本MVPは短期間のハッカソン用アプリである。公式データの更新頻度は低く、ハッカソン期間中に定期更新、差分検査、承認およびロールバックまで自動化する価値は、中心機能の実装より低い。

一方、利用した公開データの出典と時点が分からない状態にはできない。

## 決定

- ハッカソン版では、開発時に取得・変換した固定データスナップショットを使用する。
- 公式データの更新確認、定期取得、自動変換、自動公開およびロールバック運用はMVPに含めない。
- 各スナップショットには、出典、データセット名、基準年度、公式データ取得日、アプリ内データ版および変換処理版を記録する。
- データ詳細画面に、固定スナップショットの時点と注意事項を表示する。
- ブラウザキャッシュと判定結果には、固定スナップショットのデータ版を使用する。
- 取り込み処理は再実行可能なスクリプトとして残すが、定期実行基盤は作らない。

## 結果

- ハッカソン期間では、地点調査、比較、地図およびAI要約の完成を優先できる。
- 公式データ更新後も、ハッカソン版の結果は自動的には更新されない。
- 継続運用する場合は、更新検知、差分レポート、承認公開、旧版保持および復旧手順を別フェーズで設計する。
- 固定データであっても、変換件数、カバレッジ、属性値および代表地点の検証は初回取り込み時に必要になる。
