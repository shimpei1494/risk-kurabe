# ADR-0006: 住所候補の選択を地点確定操作にする

- 状態: 採用
- 決定日: 2026-07-17

## コンテキスト

災害リスクは確定した座標に対して判定するため、自由入力された住所文字列だけでは比較を開始できない。一方、地点ごとに追加の確定ボタンや確認ダイアログを設けると、気軽に比較できる体験を損なう。

住所検索結果と実際の建物位置がずれる場合もあるため、ユーザーが地図上で位置を修正できる必要がある。

## 決定

- ユーザーが住所検索の候補を選ぶと、地図を該当位置へ移動してピンを表示する。
- 住所候補を選んだ時点で、その比較地点を確定済みとして扱う。
- ユーザーは確定後のピンをドラッグして位置を修正できる。
- ピンを修正した場合は、入力住所付近でユーザーが指定した地点として扱う。
- 自由入力しただけで住所候補を選んでいない地点は、比較を開始できない。
- 各比較地点に、住所検索結果の座標かユーザーが修正した座標かを記録する。
- 追加の「位置を確定」ボタンや確認ダイアログは設けない。

## 結果

- 少ない操作で比較を開始できる一方、地点判定には常に確定座標を使用できる。
- UIでは確定済み、未確定および手動修正済みを識別できる必要がある。
- ピンを移動した場合、元の住所表示だけで正確な判定地点を表していると誤解させない表示が必要になる。
- 最近使った地点には確定座標と手動修正の有無を保存する。
- テストには、候補選択前、候補選択後、ピン移動後および再検索後の状態遷移を含める。
