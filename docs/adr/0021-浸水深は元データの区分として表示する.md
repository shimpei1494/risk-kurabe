# ADR-0021: 浸水深は元データの区分として表示する

- 状態: 採用
- 決定日: 2026-07-17

## コンテキスト

A31aとA53が提供する浸水深は地点の正確な実測値ではなく、`0.5m以上3.0m未満`などの区分である。区分の中央値などを単一の数値として表示すると、元データにない精度を持つ値だと誤解される。

また、同じ地点で複数河川・水系のポリゴンが重なる場合、3段階と6段階など粒度の異なる浸水深区分を一貫した規則で比較する必要がある。

## 決定

- ユーザー向けの名称は`浸水深区分`とし、元データが定義する区分ラベルを表示する。
- 区分の中央値、代表値または擬似的な小数値を生成しない。
- データ加工時に、元の区分コードとラベルに加えて、比較用の下限値と上限値をメートル単位で正規化する。
- 上限のない区分は、比較上の上限を正の無限大として扱う。
- 重複する区分は、上限値が大きい順に比較し、上限値が同じ場合は下限値が大きい区分を主結果にする。
- 比較規則でも一意に決まらない場合は、安定したデータセット識別子と地物識別子で順序を固定する。ただし同順位の全区分を根拠情報に残す。
- 主結果に採用しなかった河川、水系および浸水深区分も根拠情報に保持する。
- A31aの想定最大規模とA53の各年超過確率は別指標とし、互いの浸水深区分を使って優劣や順位を計算しない。

## 表示要件

- 主結果では、元データの区分ラベルと降雨条件を併記する。
- `3.0m以上`のように上限がない区分を、固定値の`3.0m`とは表示しない。
- 区分の粒度が水系によって異なる場合は、詳細な根拠欄で各水系の元区分を確認できるようにする。
- AI要約には正規化後の下限・上限だけでなく、ユーザーへ表示する元区分ラベルを渡し、単一の推定浸水深を生成させない。

## 結果

- 元データ以上の精度を装う表示を避けられる。
- 粒度の異なる浸水深区分が重なっても、主結果を決定的に選択できる。
- 上限値優先は、その区分で想定される最大側を採用する保守的な規則になる。
- 正規化表と元コードの対応が誤ると結果へ直接影響するため、全区分コードの変換テストが必要になる。
