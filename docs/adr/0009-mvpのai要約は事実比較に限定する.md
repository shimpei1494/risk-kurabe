# ADR-0009: MVPのAI要約は事実比較に限定する

- 状態: 採用
- 決定日: 2026-07-17

## コンテキスト

AI要約は、複数地点の公開データを一般ユーザーが読みやすい形へ整理する。一方、災害種別をまたいだ重み付けや住まいの総合順位にはユーザーの価値観が必要であり、MVPではその価値観を収集しない。

将来的には、ユーザーが重視条件を保存し、その観点から候補を比較して推奨理由を説明する機能を目指す。

## 決定

- MVPでは重視条件の入力、保存および選好ベース推奨を実装しない。
- 将来機能のための空画面、未使用のデータベースまたは未使用の評価処理を先行実装しない。
- 公開データによる地点判定とAIによる説明を分離し、将来の評価処理を追加しても地点判定を変更せずに済む構造にする。

MVPのAI要約では、次を許可する。

- 同じ指標について、公表値または公表区分に基づいて地点間を比較する。
- 公開データのランク名をそのまま記載する。
- `区域外`、`未公開`、`対象外`および`判定不能`を明示する。
- 要約から元の比較項目と根拠情報を確認できるようにする。

MVPのAI要約では、次を禁止する。

- 「安全」「危険」「おすすめ」「住むべき」などの居住判断を結論として示す。
- 独自の総合順位またはスコアを作る。
- 異なる災害種別を独自に重み付けする。
- 根拠データにない原因、確率または将来予測を補う。

## 将来の方向性

- ユーザーが重視条件と優先度を保存できるようにする。
- 重視条件に対する候補の評価は、AIの自由回答だけで決めず、入力と計算過程を追跡できる評価規則で算出する。
- AIは算出済みの比較結果を説明し、どの条件が推奨に影響したか、データが不足している箇所はどこかを明示する。
- 将来の推奨は「客観的に最も安全」ではなく、「指定された重視条件に最も合う候補」と表現する。
- 重視条件の保存、編集、削除およびAIへの送信について、ユーザーが制御できるようにする。

## 結果

- MVPのAI要約は、公開データの説明責任を保った補助機能になる。
- 将来の選好ベース推奨を追加しても、公開データによる地点判定とユーザー価値観による評価を区別できる。
- AI出力には、禁止表現、根拠外の値、欠落したデータ状態および不正な順位付けを検出する検証が必要になる。
