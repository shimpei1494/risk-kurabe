# ADR-0011: MVPは1地点から調査可能にする

- 状態: 採用
- 決定日: 2026-07-17

## コンテキスト

引越し候補が1件に絞られているユーザーは、その地点の災害リスクを確認して問題がなければ、別地点を比較する必要がない。最初から2地点以上の入力を要求すると、地点の情報だけを知りたいユーザーへ不要な操作を求めることになる。

## 決定

- MVPは1地点から災害リスクを調査できるようにする。
- 1地点では、洪水、適用可能な地域追加リスク情報、データ状態、境界警告、地図および根拠情報を表示する。
- 単地点結果から「地点を追加して比較」を実行できるようにする。
- 2地点目を追加すると、同じ結果画面を地点比較へ切り替える。
- 最大3地点まで追加できる。
- 地点比較は複数の地点調査を並べる機能とし、同じ地点判定結果を再利用する。
- AI要約は1地点でも利用できる。1地点では事実説明、2地点以上では同じ指標について事実比較を行う。

## 主要な操作文言

- 初回: `この地点のリスクを見る`
- 単地点結果: `地点を追加して比較`
- 2地点以上: `比較結果`
- 3地点登録済み: `地点を入れ替える`

## 結果

- プロダクトの基本単位は地点比較ではなく地点調査になる。
- 結果画面は1〜3地点に応じて表示構造を切り替える必要がある。
- ルーティングおよび状態管理は、単地点結果と比較結果を別のデータモデルに分けず、地点数の違いとして扱う。
- 住所検索、地点判定および根拠情報の処理は、1地点目から実行可能にする。
- テストには1地点、2地点、3地点、地点追加および地点入れ替えの遷移を含める。
