# ADR-0010: MVPは匿名の利用状況だけを計測する

- 状態: 採用
- 決定日: 2026-07-17

## コンテキスト

MVP公開後に、比較を完了できているか、AI要約が利用されているか、どの処理で失敗しているかを把握する必要がある。一方、本アプリは住所と災害リスクを扱うため、利用状況の把握を目的として個々の住所や比較内容を収集すべきではない。

## 決定

- ページ表示と性能の把握にはCloudflare Web Analyticsを使用する。
- アプリ固有の集計イベントにはWorkers Analytics Engineを使用する。
- 次の回数と処理時間を集計する。
  - 1地点の調査開始回数と完了回数
  - 住所候補を2地点以上確定した回数
  - 比較の開始回数と完了回数
  - 単地点結果から地点追加を開始した回数
  - 比較完了までの処理時間
  - AI要約の実行、成功および失敗回数
  - 住所検索と地点判定のエラー種別
- 結果画面に「この情報は役に立ちましたか？」という二択の評価を設け、回答を集計する。
- 自由記述のフィードバックはMVPに設けない。
- 分析イベントに住所、座標、入力文字列、地点ごとの判定値、ユーザーまたは端末の識別子を含めない。
- 分析イベントを、端末内に保存した最近使った地点と関連付けない。

## 結果

- 個々の利用者や比較地点を追跡せず、MVPの利用状況と障害傾向を把握できる。
- 継続利用を個人単位で測定することはできない。
- イベント名、許可フィールドおよび禁止フィールドをコード上で定義し、任意のオブジェクトを分析基盤へ送れないようにする。
- テストで、分析イベントに禁止フィールドが含まれないことを検証する。
