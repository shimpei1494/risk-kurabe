# ADR-0014: URLフラグメントで簡易共有する

- 状態: 採用
- 決定日: 2026-07-17

## コンテキスト

調査・比較した地点を他の人へ簡単に共有したい。一方、MVPではアカウント、サーバー上の履歴および永続データベースを導入しない。また、通常操作だけで住所や座標がURL、サーバーログまたは分析イベントへ露出することは避けたい。

## 決定

- DB不要の簡易共有リンクをMVPへ含める。
- 通常の調査URLには住所と座標を含めない。
- 結果画面でユーザーが `共有リンクを作成` を実行した場合にだけ共有リンクを生成する。
- 共有情報は、形式の版、最大3地点の確定座標、地点の順番およびリンク作成日だけに限定する。
- 住所、ユーザーが編集した地点名、地点判定結果およびAI要約を共有リンクへ含めない。
- 共有情報はクエリ文字列ではなくURLフラグメントへ格納する。
- 共有リンクの作成前に、リンクを知っている人が地点を確認できることと、リンクを後から無効化できないことを説明する。
- 共有先では、リンク内の座標を現在のデータ版と判定ロジックで判定する。
- 共有リンクを開いただけでは、地点を `最近使った地点` に保存しない。受信者が地点調査または比較を実行した時点で保存する。
- 分析イベントにURLフラグメントまたは共有座標を含めない。
- フラグメントの版、地点数、座標範囲、数値精度および全長を検証し、不正なリンクを安全に拒否する。

## 結果

- 永続データベースなしで地点を共有できる。
- URLフラグメントは通常のHTTPリクエスト、サーバーログおよびRefererへ送信されないが、リンクを受け取ったサービスと相手には座標が分かる。
- 共有リンクの取り消し、アクセス期限、閲覧制限、地点名および過去の判定結果の固定は提供できない。
- 将来これらが必要になった場合は、ランダムな共有IDとサーバー保存を使う共有方式を別途設計する。
- テストには、1〜3地点、改ざん、範囲外座標、非数値、過剰な桁数、未知の版、過長リンクおよび旧版互換を含める。
