# ADR-0016: ハッカソン版の対象地域を関東にする

- 状態: 採用
- 決定日: 2026-07-17
- 置換対象: ADR-0001

## コンテキスト

当初はMVPから全国対応する方針だったが、開発期間は2週間、開発者は1人である。想定最大規模の洪水データに加え、ダウンロード・前処理が必要な頻度別の多段階浸水想定データを全国分扱うと、データ基盤の構築が中心機能の完成を妨げる。

本アプリは都知事杯オープンデータ・ハッカソンへ提出するため、東京都オープンデータを利用した都民向け価値を明確に示す必要がある。

## 決定

- ハッカソン版の地点調査対象を、東京都、神奈川県、埼玉県、千葉県、茨城県、栃木県および群馬県の関東1都6県に限定する。
- 関東では、想定最大規模の洪水浸水想定区域データを使用する。
- 関東では、公開データが存在する水系について頻度別の多段階浸水想定データを使用する。
- 東京都内では、東京都オープンデータの地域危険度を追加し、建物倒壊危険度と火災危険度を表示する。
- 東京都外の関東6県では、東京都地域危険度を `対象外` と表示する。
- 関東外の住所を検索した場合は、ハッカソン版の調査対象外であることを説明し、災害リスク判定を実行しない。
- 全国対応は、継続運用フェーズでデータ処理と配信方式を検証した後に行う。

## オープンデータの位置付け

- 東京都の地域危険度データを、東京都内の地点調査および比較結果の中核情報として使用する。
- 国土交通省のA31aおよびA53を組み合わせ、東京都データ単独では得られない洪水と地震の横断的な比較を提供する。
- データ詳細に、各データセットの名称、提供者、基準年度、取得日、利用条件および注意事項を表示する。

## 結果

- 全国データ処理より、関東での地点判定、比較体験、根拠表示および東京都オープンデータ活用の完成度を優先できる。
- UI、共有リンクおよび住所候補には、ハッカソン版の対応地域を明示する必要がある。
- 将来の全国対応に備え、地域固有データと共通洪水データの契約は分離するが、全国分のデータや未使用コードは先行作成しない。
