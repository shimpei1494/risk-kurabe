# ADR-0002: ピン地点を主結果にして境界を警告する

- 状態: 採用
- 決定日: 2026-07-17

## コンテキスト

住所検索で得た座標と災害リスクのポリゴンには位置精度の限界がある。また、ピンが浸水深区分などの境界付近にある場合、わずかな位置の違いで判定が変わることがある。

ピン周辺の最大値を常に主結果にすると危険側へ過度に寄せる一方、ピン一点の値だけを示すと精度以上に確実な判定だと誤解される可能性がある。

## 決定

- 確定したピン地点の値を主結果として表示する。
- ピン周辺に主結果と異なる判定が存在する場合は境界警告を表示する。
- 境界検査はリスク指標ごとに行い、ピンから半径25m以内を検査範囲とする。
- 検査範囲内に、その指標の主結果と異なる値または区域外の領域が存在するときに境界警告を付ける。
- 境界警告は、異なる判定が周辺にあることと、ピン位置を確認できることを伝える。
- 境界警告によって主結果を周辺の最大値に置き換えない。
- 周辺の値は判定根拠として保持するが、主結果より強く強調せず、地図で位置関係を確認できるようにする。
- 基本の警告文は `この地点は区域の境界付近です。ピンの位置によって判定が変わる可能性があります。` とする。

## 結果

- 比較表では、地点判定と境界警告を別の情報として表示する必要がある。
- 地図では、ユーザーがピンとリスク領域の位置関係を確認できる必要がある。
- データ取得処理は、ピン一点の照合に加えて周辺領域の差異を検査できる必要がある。
- テストには、領域内、領域外、境界上、境界付近、複数領域が重なる地点を含める。
- 半径25mは、A31aの位置精度を基準にしたハッカソン版の固定値とする。データ種別やジオコーダの検証結果に応じた距離調整は将来の見直し対象とする。
