# ADR-0004: カバレッジ情報でデータ状態を判定する

- 状態: 採用
- 決定日: 2026-07-17

## コンテキスト

比較地点がリスク領域のポリゴンに含まれない場合でも、それだけでは公開データ上の区域外なのか、対象地域のデータが未収録または処理不能なのかを区別できない。「データなし」を安全と誤認させないため、この違いを機械的に判定する必要がある。

また、想定最大規模の洪水浸水想定区域データと、頻度別の多段階浸水想定データでは収録範囲が異なる。

## 決定

- 公開データの取り込み時に、地域、河川、データ種別、基準年度、取得日および処理成否をカバレッジ情報として記録する。
- 対象データの収録を確認でき、ピンがどの対象ポリゴンにも含まれない場合は、`公開データ上、想定最大規模の洪水浸水想定区域外` と表示する。
- 必要な対象データが未収録、または取り込み・判定処理に失敗した場合は、`判定データなし` と表示する。
- 想定最大規模を判定できても、対象地点に適用できる多段階浸水想定データが収録されていない場合は、`頻度別データは公開されていません` と表示する。
- カバレッジ情報自体が不明な場合は、安全側の値を推測せず `判定データなし` とする。

## 結果

- 地点判定の値だけでなく、判定に利用できたデータ範囲も問い合わせる必要がある。
- データ更新処理には、ファイルの取得だけでなくカバレッジ情報の生成と整合性検査が必要になる。
- 比較結果には、使用したカバレッジ情報の版を根拠情報として関連付ける。
- テストには、収録済み区域内、収録済み区域外、未収録地域、処理失敗およびカバレッジ不明を含める。
