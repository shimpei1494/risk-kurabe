# ADR-0020: 地図には選択中の1指標だけを表示する

- 状態: 採用
- 決定日: 2026-07-17

## コンテキスト

ハッカソン版は、想定最大規模の洪水、複数の降雨規模に対応する頻度別洪水、東京都の建物倒壊危険度および火災危険度を扱う。これらを同じ地図へ同時に重ねると、色、凡例および区域の意味が混ざり、どの指標を見ているか分かりにくくなる。

比較では最大3地点の位置関係を保ちながら、結果表で注目している指標の空間的な根拠を確認できる必要がある。

## 決定

- 地図上に表示する主題リスクレイヤーは常に1指標だけとする。
- 初期表示はA31aの想定最大規模洪水浸水深とする。
- 結果表で指標を選択すると、地図の選択中指標も連動して切り替える。
- A53は、選択した年超過確率のレイヤーだけを表示する。
- 東京都内の地点がある場合は、建物倒壊危険度と火災危険度も選択できる。
- 選択中の指標に対応する凡例、出典、データ基準時点およびデータ状態を地図付近に表示する。
- 比較中の地点ピンは、選択中指標の値や対象範囲にかかわらず最大3件すべてを常時表示する。
- 複数の主題リスクレイヤーを重ね合わせる機能はMVPへ含めない。

## 操作と状態

- 選択中指標は画面内の一時的なUI状態とし、サーバーまたはlocalStorageへ保存しない。
- 選択した指標が一部地点で`未公開`または`対象外`でも、その地点のピンは隠さない。
- 選択した指標が全地点で利用できない場合も、空のリスクレイヤーと理由を表示し、別の指標を自動選択してユーザーの選択を上書きしない。
- 地図レイヤーの切り替えによって地点判定を再実行せず、既に得た地点調査結果を利用する。

## 結果

- 指標ごとに異なる色尺度と凡例を明確に分離できる。
- 比較表から地図上の根拠へ移る操作が一貫する。
- A53の複数降雨規模を同時描画する複雑さを避けられる。
- ユーザーが複数災害の重なりを一枚で確認する用途は、MVP後の検討事項になる。
