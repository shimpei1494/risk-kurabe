# ADR-0013: 地点判定をデータ版付きでタブ内キャッシュする

- 状態: 採用
- 決定日: 2026-07-17

## コンテキスト

災害リスクの元データは頻繁には更新しない予定であり、同じ地点と同じデータ版に対する地点判定は決定論的である。ページ再読み込みのたびに同じ判定を実行すると、応答時間と処理量を不要に増やす。

一方、元データまたは判定ロジックが更新された後に古い結果を再利用すべきではない。

## 決定

- 成功した地点判定を `sessionStorage` に保存し、同じタブ内で再利用する。
- キャッシュキーには、確定座標、データセット版および判定ロジック版を含める。
- 復元時に軽量なデータ版情報を確認し、版が一致する場合だけキャッシュを利用する。
- データセット版または判定ロジック版が変わった場合は再判定する。
- ピンを移動した場合は、移動後の座標で再判定する。
- `区域外`、`未公開`、`対象外`および正常に確定した`判定不能`はキャッシュできる。
- 通信エラー、タイムアウトおよび一時的な処理失敗はキャッシュしない。
- 境界警告、根拠情報およびカバレッジ情報の版を判定結果と一緒に保存する。
- 地点判定は `localStorage` およびサーバーの永続データベースへ保存しない。
- サーバー側でキャッシュする場合も住所をキーにせず、座標、データ版および判定ロジック版を使用する。

## 結果

- 同じタブ内の再読み込みや画面遷移では、不要な再判定を避けられる。
- データ更新後は版の不一致によって自動的に再判定される。
- キャッシュ形式にはスキーマ版を持たせ、読めない旧形式を安全に破棄する必要がある。
- テストにはキャッシュ命中、各版の不一致、ピン移動、一時エラーおよび破損キャッシュを含める。
