# ADR-0003: 重複する洪水判定では最大浸水深を表示する

- 状態: 採用
- 決定日: 2026-07-17

## コンテキスト

同じ比較地点が、複数河川の洪水浸水想定区域に含まれる場合がある。単一の値だけを無条件に選ぶと、より深い浸水想定を見落としたり、どの河川のデータを参照したか分からなくなったりする。

## 決定

- 同じピンに複数河川の洪水ポリゴンが重なる場合、最も深い浸水深を地点判定の主結果として表示する。
- 該当した全河川および全ポリゴンの根拠情報を保持する。
- データ詳細では主結果を決めた河川を識別可能にし、その他の該当河川も確認できるようにする。
- AIによる要約を行う場合も、主結果と根拠情報の範囲を越えた推測をさせない。

## 結果

- 比較画面では、見落としを避けるための代表値を簡潔に表示できる。
- ユーザーはデータ詳細で、代表値以外の該当河川も確認できる。
- 地点判定処理は、最初に一致したポリゴンで終了せず、同種の候補をすべて収集してから主結果を選ぶ必要がある。
- テストには、異なる浸水深の重複、同じ浸水深の重複、同一河川内の重複を含める。
