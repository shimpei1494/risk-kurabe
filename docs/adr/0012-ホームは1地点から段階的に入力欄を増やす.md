# ADR-0012: ホームは1地点から段階的に入力欄を増やす

- 状態: 採用
- 決定日: 2026-07-17

## コンテキスト

MVPは1地点から利用できる。ホームに3地点分の入力欄を常時表示すると、単地点を調べたいユーザーへ複数入力が必要だと誤解させる。一方、最初から比較したいユーザーが結果画面を経由しなければならない設計も避けたい。

## 決定

- ホームの初期状態では住所入力欄を1つだけ表示する。
- 1地点を確定した後の主操作を `この地点のリスクを見る` とする。
- ホームに `比較する地点を追加` を設け、2件目と3件目の入力欄を順に表示できるようにする。
- 単地点結果にも `地点を追加して比較` を設ける。
- 各地点名の初期値を追加順に `地点1`、`地点2`、`地点3` とする。
- 地点名は任意で編集できるが、住所、座標および地点名をAI要約入力へ含めない。
- AI要約では内部識別子を使用し、ブラウザ側でユーザーの地点名へ置き換える。

## 結果

- 単地点調査が最短の利用経路になる。
- 最初から比較したいユーザーは、ホームから最大3地点を入力できる。
- ホーム、単地点結果および比較結果で、同じ地点入力UIを再利用する必要がある。
- UIには地点追加、地点削除、地点名編集および最大件数到達の状態が必要になる。
- テストには、1件だけの初期状態、2件目と3件目の段階表示、途中削除および地点名編集を含める。
