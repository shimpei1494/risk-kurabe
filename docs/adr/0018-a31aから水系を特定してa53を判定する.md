# ADR-0018: A31aから水系を特定してA53を判定する

- 状態: 採用
- 決定日: 2026-07-17

## コンテキスト

A31aの想定最大規模浸水情報とA53の頻度別浸水情報は、どちらも地点の洪水リスクを説明するが、収録範囲は同一ではない。A53は水系や降雨規模によって公開状況が異なるため、単に地点がA53ポリゴンに一致しなかったことを「浸水区域外」と表示すると、未公開データを区域外と誤認させる。

また、同じ地点が複数河川・水系の洪水ポリゴンに含まれる場合があり、主結果と根拠情報の扱いを統一する必要がある。

## 決定

- 最初にA31aで地点に一致する全河川を取得し、それらの水系をA53の探索候補にする。
- A31aで一致した水系について、データマニフェストとカバレッジ情報からA53の公開・収録状況を確認する。
- A53は、収録済みの水系と降雨規模だけを地点照合する。
- 地点がA53ポリゴンに一致した場合は、その降雨規模の浸水深を表示する。
- 同じ降雨規模で複数水系のA53ポリゴンに一致した場合は、最も深い浸水深を主結果にし、該当した全水系を根拠情報に残す。
- A53ポリゴンに一致しなかった場合は、カバレッジ情報に基づいてデータ状態を判定する。

## A53未一致時の表示状態

- 対象水系・降雨規模のデータを収録済み: `公開データ上、この降雨規模の浸水区域外`
- 対象水系にA53が公開されていない: `この水系の頻度別データは公開されていません`
- A31aに一致する河川がなく水系を特定できない: `頻度別浸水の判定データなし`
- 変換失敗や成果物不整合などで確認できない: `判定不能`

複数の候補水系で公開状況が異なる場合は、収録済みデータの判定を表示したうえで、未公開の水系も根拠情報に明記する。未公開の水系があることを理由に、収録済み水系の結果を捨てない。

## データ加工要件

- A31aとA53の属性から、比較可能な正規化済み水系識別子を生成する。
- 水系名だけの文字列一致に依存せず、利用可能なコードを優先し、コードがない場合の対応表を版管理する。
- カバレッジ情報には、水系識別子、降雨規模、公開元、収録成否および変換処理結果を保持する。
- A31aからA53へ関連付けられない河川は変換時に検出し、件数と対象を検証レポートへ出力する。

## 結果

- A53の未公開と浸水区域外を区別して表示できる。
- 頻度別情報はA31aで特定した水系に限定され、無関係な水系のポリゴンを根拠にしない。
- 同地点に複数水系が重なっても、保守的な主結果と追跡可能な全根拠を両立できる。
- A31aに一致しない地点ではA53を独立探索しないため、A31aとA53の形状差によってA53だけに一致し得るケースは表示対象外になる。この制約はデータ検証時に件数を確認し、必要なら将来ADRを見直す。
