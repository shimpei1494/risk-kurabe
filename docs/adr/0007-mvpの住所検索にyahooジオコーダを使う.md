# ADR-0007: MVPの住所検索にYahoo!ジオコーダを使う

- 状態: 採用
- 決定日: 2026-07-17

## コンテキスト

全国の住所を確定座標へ変換する必要がある。デジタル庁のABRジオコーダーは、住所を第三者へ送らず自前で運用できる一方、全国データを用いたSQLite検索基盤の構築と運用が必要であり、通常のCloudflare Workersへそのまま配置できない。

MVPでは、住所検索基盤の構築より災害リスク比較の価値検証を優先する。

## 決定

- MVPの住所検索にはYahoo!ジオコーダAPIを使用する。
- ブラウザからYahoo!へ直接アクセスせず、TanStack Startのサーバー関数を経由する。
- Yahoo!へ入力住所を送信すること、送信目的および保存方針を画面上で説明する。
- アプリのサーバーログおよびサーバーキャッシュに入力住所を保存しない。
- Yahoo!固有のレスポンスはサーバー側でアプリ共通の住所候補へ変換し、UIおよび地点判定から分離する。
- アプリ共通の住所候補には、表示住所、座標、検索一致レベルおよびプロバイダー識別子を含める。
- 将来、ABRジオコーダーの精度、全国版DB容量、応答性能および運用費を検証し、条件を満たす場合は置き換えを検討する。

## 結果

- MVPの住所検索を短期間で実装できる。
- 入力住所はYahoo!へ送信されるため、利用者への説明と適切な同意が必要になる。
- APIキーをブラウザへ公開せず、サーバー側でレート制限およびエラー変換を行える。
- UIはYahoo!固有のフィールドやエラーへ依存しない。
- ABRへ移行するときは、住所検索の実装と運用基盤を交換し、同じアプリ共通契約に対する適合テストを通す。

## ABR移行前の検証項目

- 全国版SQLite DBの容量と構築時間
- 都市部、地方、住居表示地域および地番地域での一致レベルと座標精度
- Yahoo!ジオコーダーと同じ検証住所を使った結果比較
- Cloudflare Containersまたは別の実行基盤でのメモリ、応答時間、コールドスタートおよび費用
- データ更新手順と障害時の復旧手順
