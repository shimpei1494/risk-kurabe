# ADR-0008: AI要約はユーザー操作時だけ生成する

- 状態: 採用
- 決定日: 2026-07-17

## コンテキスト

AI要約は公開データを読みやすく整理する補助機能であり、地点判定そのものではない。比較結果を表示するたびに自動生成すると、要約を読まない利用者についてもOpenAI APIの利用量が発生する。

また、AI APIの障害や利用上限によって、災害リスク比較の本体が利用できなくなることは避ける必要がある。

## 決定

- 比較結果の表示時にはAI要約を自動生成しない。
- ユーザーが「AIで要約」ボタンを押した場合にだけ生成する。
- 1〜3地点分をまとめ、ユーザーの実行1回につき1リクエストで生成する。
- 1地点ではその地点の公開データを説明し、2地点以上では同じ指標について地点間の事実比較を行う。
- 公開データによる比較表と地図を先に表示し、AI要約は非同期で生成する。
- 生成中も比較表と地図を操作可能にする。
- AI要約の失敗、タイムアウトまたは利用上限到達によって、比較表と地図を利用不能にしない。
- 調査地点、判定結果、AIモデル、プロンプト版および出力スキーマ版が変わらない限り、同じタブ内で生成済みの要約を再利用する。
- 生成済みの要約は `sessionStorage` に保存し、ページ再読み込み後も同じタブ内で再利用する。
- 要約のキャッシュキーに住所、座標およびユーザーが編集した地点名を直接含めず、AI要約入力の正規化済みハッシュと生成設定の版を使用する。
- AI要約は `localStorage`、端末内の最近使った地点およびサーバーへ永続保存しない。
- OpenAIへはAI要約入力だけを送信し、住所と座標は送信しない。

## 結果

- OpenAI APIの利用量を、要約を希望するユーザーに限定できる。
- AI APIが利用できない場合でも、アプリの中心機能を提供できる。
- UIには未生成、生成中、生成済み、キャッシュから復元済み、再生成が必要および生成失敗の状態が必要になる。
- サーバー側には入力検証、出力検証、タイムアウト、レート制限および安全なエラー変換が必要になる。
