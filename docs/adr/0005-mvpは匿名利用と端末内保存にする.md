# ADR-0005: MVPは匿名利用と端末内保存にする

- 状態: 採用
- 決定日: 2026-07-17

## コンテキスト

MVPでは、ユーザーが登録作業をせず気軽に災害リスク比較を試せることを優先する。一方、住所など繰り返し比較に使う入力は再利用できると便利である。

比較履歴や共有URLは将来的に有用だが、MVPから実装すると認証、データベース、住所情報の管理および削除対応が必要になる。

## 決定

- MVPにはアカウントおよびログイン機能を設けない。
- 比較地点や比較履歴をサーバーへ永続保存しない。
- 最近使った地点は、ユーザーのブラウザ内のストレージに保存する。
- 端末内に保存した地点をユーザーが確認・削除できるようにする。
- 比較を実行した時点で、使用した地点を自動保存する。
- 最近使った地点は最大10件とし、最終利用から90日経過した地点を自動削除する。
- 保存内容は、住所表示名、確定座標および最終利用日時に限定する。比較結果とAI要約は保存しない。
- 入力候補から地点を個別削除できるようにし、端末内の保存データを一括削除する手段も用意する。
- 入力した地点が利用中の端末にだけ保存されることを画面上で説明する。
- 座標をURLフラグメントへ格納するDB不要の簡易共有リンクをMVPに含める。詳細はADR-0014に従う。
- 期限付き、取り消し可能、地点名付きなどサーバー保存を必要とする共有、サーバー上の履歴および端末間同期はMVP後に検討する。
- 将来の認証機能追加を想定するが、MVPでは認証やサーバー保存のための未使用の抽象化を先行実装しない。

## OpenAIへの送信

- AI要約に住所や座標が不要である限り、それらをOpenAIへ送信しない。
- MVPでは、地点判定後の構造化されたリスク情報だけをAI要約入力とする。
- 地点は住所ではなく「現在地」「候補A」「候補B」などの役割で識別する。
- 将来、住所または座標が必要なAI機能を追加する場合は、送信目的、表示上の説明およびデータ取り扱いを改めて判断する。

## 結果

- MVPではCloudflare D1などの永続データベースを必要としない。
- 住所検索および災害リスク判定のリクエストは処理に必要だが、アプリのデータベースには保存しない。
- 同じブラウザでは地点を再利用できる。簡易共有リンクで座標を別端末へ渡せるが、履歴の同期および復旧はできない。
- 共有端末では保存した住所を他の利用者が見られる可能性があるため、保存の説明と削除手段が必要になる。
